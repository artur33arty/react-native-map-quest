# Отчет по проекту «Интерактивная карта с базой данных SQLite и GPS-квестами»

## 1. Введение

Разработанное мобильное приложение представляет собой интерактивную геоинформационную систему, созданную на базе React Native и Expo. Основная цель проекта — обеспечить пользователю возможность работы с картой, сохранения геометок, прикрепления изображений, а также организации игрового GPS-квеста с использованием геолокации и уведомлений. Приложение ориентировано на работу в мобильной среде и протестировано через клиент Expo Go.

## 2. Цель и задачи проекта

🎯 **Цель**

Создание мобильного приложения с интеграцией карты, локальной базы данных и GPS-трекинга для реализации интерактивных гео-квестов.

📌 **Основные задачи:**

- реализация интерактивной карты;
- добавление и хранение геомаркеров;
- интеграция фото из галереи устройства;
- создание локальной БД SQLite;
- реализация GPS-трекинга пользователя;
- внедрение системы уведомлений;
- разработка игрового режима «квест».

## 3. Технологический стек

Проект реализован с использованием следующих технологий:

- **React Native** — мобильный фреймворк
- **Expo (SDK 54)** — среда разработки
- **expo-router** — файловая навигация
- **react-native-maps** — работа с картами
- **expo-sqlite** — локальная база данных
- **expo-location** — GPS-геолокация
- **expo-notifications** — пуш-уведомления
- **expo-image-picker** — доступ к галерее

## 4. Архитектура приложения

Приложение построено по модульной архитектуре с разделением ответственности:

- **UI слой** — интерфейсы экранов
- **Data Access Layer** — работа с SQLite
- **Context Layer (React Context)** — глобальное состояние

Такой подход обеспечивает:
- изоляцию логики;
- удобство масштабирования;
- упрощение тестирования;
- повторное использование кода.

## 5. Навигация и интерфейс

Навигация реализована через `expo-router`:

- `app/index.tsx` — главный экран карты
- `app/marker/[id].tsx` — экран деталей маркера

Основной функционал интерфейса:
- интерактивная карта;
- добавление маркеров (Long Press);
- просмотр и редактирование точек;
- прикрепление изображений из галереи.

## 6. Работа с изображениями

Для добавления фотографий используется `expo-image-picker`.

**Функциональность:**
- выбор изображения из галереи;
- получение URI файла;
- сохранение пути в SQLite;
- отображение изображения на экране маркера.

> ⚠️ **Ограничение:** при удалении фото из галереи устройства ссылка становится недействительной.

## 7. Локальная база данных SQLite

📊 **Структура базы данных**

Созданы две связанные таблицы:

```sql
-- Таблица маркеров
CREATE TABLE IF NOT EXISTS markers (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    latitude REAL NOT NULL,
    longitude REAL NOT NULL,
    title TEXT
);

-- Таблица изображений маркеров
CREATE TABLE IF NOT EXISTS marker_images (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    marker_id INTEGER NOT NULL,
    uri TEXT NOT NULL,
    FOREIGN KEY (marker_id) REFERENCES markers (id) ON DELETE CASCADE
);
```
Один маркер → множество изображений

Используется связь «Один ко многим»

Включено каскадное удаление (ON DELETE CASCADE)

## 8. Геолокация и GPS-трекинг

Функциональность реализована через `expo-location`.

📍 **Возможности:**

- отслеживание текущего положения пользователя;
- отображение синей точки на карте;
- обновление координат в реальном времени.

## 9. Расчет расстояния

Для вычисления расстояния между координатами используется **формула Хаверсина**, учитывающая кривизну Земли. Она применяется для:

- определения приближения к маркерам;
- активации уведомлений;
- запуска игровых событий.

## 10. Система уведомлений

Используется модуль `expo-notifications`.

📌 **Логика работы:**

- уведомление отправляется при расстоянии < 100 метров;
- предотвращается повторная отправка;
- используется кэш (`Set` + `useRef`);
- уведомление срабатывает один раз на маркер.

## 11. Игровой режим «Квест»

Реализован демонстрационный режим автоматического прохождения маршрута.

🎮 **Алгоритм работы:**

1. Пользователь добавляет маркеры;
2. Запускает режим теста;
3. Виртуальная метка начинает движение;
4. При приближении: отправляется уведомление;
5. При достижении: маркер удаляется из SQLite;
6. Система переходит к следующей точке;
7. После завершения отображается сообщение об окончании квеста.

## 12. Обработка ошибок

Приложение устойчиво к сбоям.

⚙️ **Реализовано:**

- обработка отказа GPS и уведомлений;
- `try-catch` для всех операций SQLite;
- отображение ошибок через UI;
- защита от повторных уведомлений.

## 13. Инструкция по запуску

**Шаг 1. Установка зависимостей**

```bash
npm install --legacy-peer-deps
```
Шаг 2. Запуск проекта
```bash
 npx expo start 
```

Шаг 3. Подключение устройства

- установить Expo Go;

- подключить к одной Wi-Fi сети;

- отсканировать QR-код.

## 14. Инструкция тестирования 
Открыть приложение;

Добавить несколько маркеров;

Нажать «▶ Запустить тест GPS»;

Наблюдать:

движение виртуального объекта;

уведомления при приближении;

удаление точек;

автоматический переход между маркерами;

После завершения появится сообщение об окончании.

## 15. Ограничения системы
изображения зависят от файлов галереи устройства;

при удалении фото из телефона ссылка в БД становится невалидной;

работа GPS зависит от разрешений устройства;

возможны погрешности геолокации.
